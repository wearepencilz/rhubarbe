# Decision Log

When a change introduces a new dependency, changes architecture, sets a precedent for future work, or reverses a previous decision, append an entry to `docs/decisions.md`.

Use the next ADR number. Keep it short — 2-4 sentences per section:

```
### ADR-NNN: Title
**Date:** YYYY-MM-DD | **Status:** accepted
**Context:** Why.
**Decision:** What.
**Consequences:** What changed.
```

Not every commit needs an entry. Skip routine features, bug fixes, and styling changes.
